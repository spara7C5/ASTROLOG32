This is advice that something needs to be fixed. - this page is to tell you what to check & then what to do.
Meanwhile, Astrolog32 will function, after a fashion, without access to the Atlases nor Swiss Ephemeris files - making charts with data you put in manually, within the limitation of only accessing the backup ephemeris; you can't call any charts, interpretations nor atlas, nor is there a way of altering the program's size and functioning.
Astrolog32 has an ultimate default location of Greenwich, UK, and that timezone, UT.
But the program's on life-support: you must let it create an astrolog32.dat, or import one, ..and proceed from there (there may be more issues: later this page will answer any queries about directory-paths if you get "Cannot find Ephemeris/American/.. , and such.."- see further on how to deal with this situation).

So I can recall three possible reasons why installers of Astrolog32 see this - to narrow down:
  • At the end of the "Welcome to Astrolog32" dialog, you were invited to create a configuration file (astrolog32.dat). If you declined, this warning may appear.
  • Someone deleted the Config.file (astrolog32.dat), and this may appear - it's like a baby, it has to generate a control interface with you
  • a freak of circumstance

Even where you actually have an astrolog32.dat, you may still get: Cannot find ephemeris files, ...cannot find states.atl & the like, and these will arise when you have browsed away from the default targets of Astrolog32 (pre-V2.0x) folders when they unzipped; - its folders' re-integration is covered, together with an absence of the Config.file, below.

------------------------------

If you have installed version 1.3x or before, default unzips go to C:\Astrolog32

If you have installed version 2.0x, 'offered' install paths go to C:\Program Files\Astrolog32

------------------------------

The fix may be quick, or a long chore, and so the first thing is to snoop the situation:

  • 1. Be sure you know where your Astrolog32 folder is: if not, go "My Computer", and go 'Search'; or, navigate to the folder; note its holding folder... - # with older versions of Astrolog32 you may find two Astrolog32 folders! - one at the root, but one where you installed it elsewhere

  • 2. Open the folder that you unzipped your astrolog32.zip to, and find and open \main folder; is astrolog32.dat there? If so, open it, using Notepad

  • 3. If it's not there, then if you haven't already, launch Astrolog32 by double-clicking astrolog32.exe... then in Astrolog32's window, go [Edit] > [Edit configuration file]: certainly now you have a Config.file!

  • 4. astrolog32.dat is the Config. file: with this file open, go "Save As", and snoop what folder is offered to you. Is it the \main folder in the Astrolog32 folder you put the program in? use the "Save as" dialog box to confirm this.

One job of this file is to tell the program where \miscellaneous, \charts, & \atlas & other folders are.

  • 5. Save astrolog32.dat.

 NOTE: When saving, in the next field under 'filename', ('file as type'), keep as astrolog32.dat.

 (NOTE: Older versions of Astrolog32 would generate new folders at C:\, and these arise when the program panics on not initially finding its auxiliary folders.)

------------------------------

Where you found the /main folder, were there also the other folders there?
If not, make a note of their location.

If found, were they populated with any files you may have downloaded?

------------------------------

Testing: Keep the Config.file open and now test, using the open program 'Astrolog32':

1. In Astrolog32's charting window, go [Edit] > [Edit Mainchart], and click 'Search Atlases' - do you get:
"file 'states.list.atl' not found in Directory 'C:\astrolog32\atlas\american'"? - or,
"file 'states.list.atl' not found in Directory 'C:\Program Files\astrolog32\atlas\american'"?
If so, close these dialogs: the the folders are found, but the contents are mislaid.

2. In Astrolog32's charting window, go [File] > [Save Mainchart data..]: \charts folder should appear; go 'up a folder'; there you should find, at the very least, \main, \ephemeris, \charts, \atlas folders.

3. Open \main: change the 'file as type' field to 'All files': there should be the following visible:
seorbel.txt, four astrolog.ico's, astrolog32.dat, Helpfile.txt, history texts, watlas, and others.

4.Go 'up a folder':
Open '\atlas': open '\american': retain the 'file as type' field as 'All files'. If you have installed the atlases, Can you see over 60 *.atl files? Or, even just one or two .atl files?

5. Go up a folder:
Open '\international': retain the 'file as type' field as 'All files'. If you have installed the atlases, Can you see over 245 *.atl files? Or, even just one or two .atl files?

6. If you installed extra Swiss Ephemeris files, go 'up two folders';
double-click to open \ephemeris: are there just three *.se1 files? More?


------------------------------

If the atlas files were installed, but can't be seen: - if there is no *.se1 files in ephemeris although you installed them, the Config.file is steering the program to the wrong place: it looks like there's other folders holding your files.

Find the Astrolog32 (or other) folder which actually is populated with these files, and copy down the full address path of one file from each folder.

Now, either you shift the files to where Astrolog32 expects them, or leave them, and correct the DIRECTORIES to finish the job, by inserting edits into the Config.file's DIRECTORIES.
See the 'Customizing Defaults in the Config.file', section of the Companion, in the last part called YOUR DIRECTORIES.

>>>>>>>> If all files are present, and your astrolog32.dat steers the program to them, your Astrolog should work fine now. <<<<<<<<
